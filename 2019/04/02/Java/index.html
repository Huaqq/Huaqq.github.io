1. 数组的使用

定义数组变量

<类型>[] <名字>=new <类型>[元素个数]

int[] nums=new int[100];

数组存在安全隐患。

数组中的元素必须都为同类型；大小一旦确定就不能改变；元素的个数可以是变量。故安全隐患为数组越界的存在。

编译器不检查数组下标（索引），但运行时会报出异常。

  • 数组变量是数组的管理者而非数组本身
  • 数组必须创建出来然后交给数组变量来管理
  • 数组变量之间的赋值时管理权限的赋予
  • 数组变量之间的比较是判断是否管理同一个数组，尽管内容相同，没有直接赋值也是管理着不同数组

2.类

对象=属性+服务

封装即数据和对数据的操作放在一起

this 表示当前对象本身

类的成员变量属性为private表示这个成员变量只有在类的内部才能被访问到：

  1. 成员函数中可以调用
  2. 成员变量初始化的时候可以调用

static 变量属于类变量，不属于任何一个对象，类名.static 变量是可以访问到的而其他成员变量不能这么访问。static 函数也是如此。静态函数只能调用静态函数，只能访问静态的成员变量。

不定义public 就会默认使用friendly 来修饰，friendly 作用为只要在同一个包内的成员变量就可以访问。

一个.java文件就是一个编译单元。一个编译单元允许定义很多类，但只允许有一个public属性的类,且这个类的名字和编译单元的名字一样。

  • 没有public的类只能在它所属的包里起作用
  • 有public的类可以在包外起作用,只需要在需要用到的地方import即可

4.继承

子类extends 父类之后，将得到父类中的方法，子类可以访问父类中不包括private修饰的变量。可以访问protected 修饰的变量，即使父类子类不在同一个包内。

在子类中不仅要对自己完成初始化，还要对其父类进行初始化

super() 调用父类的无参构造函数

super(...) 调用父类的对应的有参构造函数

如果父子类中有同名变量，则子类的是子类的，父类的是父类的

super.print()子类可以如此调用父类的方法

5.多态变量

这个变量在运行的时候，在具体某一时刻，它所管理的对象的类型是会变化的

子类的对象可以被当作父类的对象来使用

  • 赋值给父类的变量
  • 传递给需要父类对象的函数
  • 放进存放父类对象的容器里
  • Java对象变量是多态的，它们能保存不止一种类型的对象
  • 可以保存声明类型的对象，或声明类型的子类的对象
  • 把子类对象赋给父类的变量时，就发生了向上造型
造型Cast

父类的对象不能赋值给子类的变量（让子类变量去管理父类对象）

不同于强制类型转换（char->int->double），对象本身并没有发生变化

向上造型是安全的，把子类对象当作父类对象来用

函数调用的绑定

当通过对象变量调用函数的时候，调用哪个函数称为绑定

静态绑定：根据变量的声明类型决定

动态绑定：根据变量的动态类型决定

在成员函数中调用其他成员函数也是通过this这个对象变量来调用的

Override覆盖关系，子类和父类中存在名称和参数完全相同的函数

如Db.java中的a.print()会根据a所管理的对象所属的类去调用这一函数
